SecuAuth AI

Complete Project Structure & Detailed Explainer

1 Project Overview

SecuAuth AI is a next-generation invisible continuous authentication layer designed specifically for the Indian securities market. It adds a silent biometric + cryptographic verification system that activates after traditional login (Password + OTP). Even if a hacker steals the user's credentials through advanced phishing or social engineering, the system verifies that the real investor is physically present using the device's front camera — without showing any extra steps to the legitimate user.

2 Complete Architecture & Workflow Diagram

Understanding the Workflow:
The diagram shows the complete end-to-end process: User enters credentials → Silent camera scan → Cryptographic signature generation → Backend verification → Final decision (Access granted or Intruder alert). All steps happen in under 1.5 seconds with zero user friction.

3 The Problem We Are Solving

90%+ of account takeovers in the securities market happen through stolen credentials. Even with Two-Factor Authentication (OTP), hackers use real-time phishing kits and man-in-the-middle attacks to intercept the OTP. Once the correct OTP is entered on the legitimate platform, the system assumes the user is genuine and grants full access. Current biometric solutions (Face ID, fingerprint) are visible, platform-dependent, and can be bypassed using backup PINs.

4 Two Delivery Methods

FOR WEB PLATFORMS

Chrome Extension

For Broker Websites (Zerodha, Groww, Upstox, Angel One, 5Paisa, etc.)

How it Works:
  • Runs completely silently in the background
  • Detects login or high-value transaction attempts
  • Uses MediaPipe Face Mesh for 468 facial landmarks
  • Generates cryptographic signature using WebCrypto API
  • Requires zero changes on existing broker websites
  • Works across Chrome, Edge, and other Chromium browsers
FOR MOBILE APPLICATIONS

Security SDK

For Broker Mobile Apps (Android & iOS)

How it Works:
  • Lightweight SDK that developers can easily integrate
  • Uses device Secure Enclave / Android Keystore
  • Native camera + advanced liveness detection
  • Simple developer APIs: startSilentAuth(), verifyLiveness()
  • Works even when the app is in background
  • Minimal impact on battery and performance

5 Step-by-Step Working Flow

01

User Enters Credentials

User enters Password + OTP normally on the broker platform.

02

Silent Edge Scan Triggers

While the page shows "Processing your request...", the front camera activates silently and captures facial landmarks using MediaPipe.

03

Private Key Signature

Facial landmarks are converted into a cryptographic signature using the device's private key stored in Secure Enclave / WebCrypto.

04

Backend Verification

The backend fetches the user's registered public key and verifies the signature using ECDSA/RSA-4096. No biometric data is ever sent to the server.

05

Decision Matrix

MATCH → Access Granted
MISMATCH → Intruder Alert + Photo Capture

7 Privacy & Security Highlights

No Biometric Data Sent to Server

Only cryptographic signatures are transmitted. The actual face image is never sent to any server.

Keys Stored Locally on Device

Private keys are stored in Secure Enclave / WebCrypto. The server only stores public keys.

8 Key Advantages

Zero Friction UX

Legitimate users experience no extra steps. Security runs completely in the background.

Proactive Intruder Capture

Unlike normal blocks, the system captures photographic evidence of the hacker.
